Genesis — the agentic engine

Greenfield architecture · all-TypeScript · walking skeleton

Genesis turns a chat message into a supervised coding-agent run. A supervisor resolves the message to a session; a runner spawns a coding-agent CLI (claude -p --output-format stream-json) inside an isolated git worktree on an ExecutionHost; a projection reducer folds the NDJSON event stream into a live phase machine the chat surface renders.

Decisions locked 2026-06-04. Greenfield standalone (not extending Hawthorne) · execution core all-TypeScript (no Rust sidecar) · durable spine Trigger.dev v4 (scaffolded, activates Phase 5) · cloud sandbox deferred (ExecutionHost.snapshot? optional; Local + VPS now, microVM Phase 4). Reuses learnings, not code from Houston, Hawthorne, caltext/midday.

Request lifecycle

A turn-ending awaiting survives a result (the HITL signal is preserved); done and blocked are absorbing — trailing post-result lines can never un-terminate a run.

The host-invariance insight

Runtime = { filesystem + bash + egress }. Local, VPS, and microVM are interchangeable behind ExecutionHost. The sharp consequence: host ownership determines persistence AND credential tier — they are one choice, not two.

kindpersistencecredential tierphase
localephemeral (this box)subscription-clean1 ✓
vpsdurable (user box)subscription-clean1 ✓
microvmsnapshot-suspend (idle ≈ free)keyed (boundary-injected)4 (deferred)

Anthropic's 2026 ToS is load-bearing here: subscription OAuth is prohibited in third-party cloud, so the keyed-credential tier is a consequence of choosing an un-owned host, not an independent knob.
