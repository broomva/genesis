5 · Migration (additive, idempotent)

Appended to MIGRATE_SQL in packages/db/src/schema.ts — safe to run repeatedly, on the live VPS DB:

ALTER TABLE sessions ADD COLUMN IF NOT EXISTS archived  boolean NOT NULL DEFAULT false;
ALTER TABLE sessions ADD COLUMN IF NOT EXISTS title     text;
ALTER TABLE turns    ADD COLUMN IF NOT EXISTS input_tokens         integer;
ALTER TABLE turns    ADD COLUMN IF NOT EXISTS output_tokens        integer;
ALTER TABLE turns    ADD COLUMN IF NOT EXISTS cache_read_tokens    integer;
ALTER TABLE turns    ADD COLUMN IF NOT EXISTS cache_creation_tokens integer;
ALTER TABLE turns    ADD COLUMN IF NOT EXISTS cost_usd             double precision;
Hard delete has no FK cascade (session_id is plain text) → deleteSession must delete turns then sessions in that order, and clear the in-flight chains-map entry + call control.reset first to avoid an orphaned live process writing phase back after the row is gone (same race the existing reset() B2 guard handles).
